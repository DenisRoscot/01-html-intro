Web-Grundlagen: Client, Server und Protokolle

1. Was ist ein Client?

Der Client ist der Dienstnehmer. Im Web-Kontext ist das meist dein Webbrowser (Chrome, Firefox, Safari) auf deinem Laptop oder Smartphone. Er ist das Werkzeug, mit dem du Anfragen stellst und Inhalte konsumierst.

2. Was ist ein Server?

Der Server ist der Dienstleister. Es handelt sich dabei um einen leistungsstarken Rechner, der rund um die Uhr läuft und darauf wartet, Daten auszuliefern. Er speichert die Dateien einer Webseite (Texte, Bilder, Code) und schickt sie auf Anfrage an den Client.

3. Was passiert bei einem Webseitenaufruf?

  1. DNS-Anfrage: Dein Browser schlägt im „Telefonbuch des Internets“ (DNS) nach, welche IP-Adresse zu dem Namen (z. B. google.de) gehört.
  2. Anfrage (Request): Der Client schickt ein digitales Paket an diese IP-Adresse mit der Nachricht: „Ich hätte gerne die Startseite.“
  3. Verarbeitung: Der Server nimmt die Anfrage entgegen, sucht die passenden Daten heraus und bereitet sie vor.
  4. Antwort (Response): Der Server schickt die Daten (HTML, Bilder etc.) zurück.
  5. Rendering: Dein Browser liest den erhaltenen Code und baut daraus die grafische Webseite zusammen, die du siehst.

4. Worin sich HTTP und HTTPS grob unterscheiden

Beide sind Protokolle – also die „Sprache“, in der Client und Server miteinander kommunizieren.

  • HTTP (Hypertext Transfer Protocol): Die Basis-Variante. Die Daten werden im Klartext übertragen. Wenn sie abgefangen werden, kann sie jeder mitlesen (unsicher bei Passwörtern).
  • HTTPS (Hypertext Transfer Protocol Secure): Die sichere Variante. Hier wird die Verbindung verschlüsselt (per SSL/TLS). Das schützt vor Lauschern und stellt sicher, dass die Webseite echt ist.
